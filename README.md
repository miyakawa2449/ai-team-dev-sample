# AI Team Development Sample

このリポジトリは、AIチーム開発の雛形プロジェクトです。

## 各AIツールの役割

- **Kiro**：仕様管理（`.kiro/steering/ai-team-roles.md` を自動読み込み）
- **Claude Code**：実装（`CLAUDE.md` を最初に読んでください）
- **Codex**：デバッグ（`CODEX.md` を最初に読んでください）
- **Human**：判断・オーケストレーション

## 目的

- AIをツールではなく「チーム」として扱う
- 役割分担による開発の安定性を示す
- reports/ に判断ログを残す運用を例示する

## 重要な考え方

- AIは万能ではない
- 判断は分散させる
- 役割を越える場合は人が判断する

```markdown
## CLAUDE.md / CODEX.md について

このプロジェクトでは、最初から  
CLAUDE.md や CODEX.md を用意していたわけではありません。

### 初期段階

- ai-roles.md による役割定義のみ
- 人間が毎回プロンプトで補足
- 小規模な修正・実験が中心

この段階では、  
**個別の AI 向け指示ファイルは不要**でした。

### 運用が進んでから起きた変化

開発が進むにつれて、次のような状況が増えてきました。

- 実装・デバッグの切り替えが頻繁になる
- 同じ注意事項を何度も伝える必要が出てくる
- 「ここから先は判断が必要」という場面が増える

### CLAUDE.md を作成した理由

Claude Code には **実装に専念してほしい** ため、

- 責任範囲
- 参照すべきドキュメント
- やってはいけないこと

を明文化しました。

これにより、  
実装の安定性と一貫性が向上しました。

### CODEX.md を作成した理由

Codex は主に **デバッグ・検証フェーズ** で使用しています。

運用が進む中で、

- バグ修正のついでに修正が広がる
- 「改善」と「仕様変更」の境界が曖昧になる

といった問題が見られたため、

- デバッグの責任範囲
- 新機能を追加しない
- 判断が必要な場合は人に戻す

というルールを CODEX.md として明文化しました。

### まとめ

- **ai-roles.md**：プロジェクト全体の憲法
- **CLAUDE.md**：実装フェーズを安定させるために導入
- **CODEX.md**：デバッグフェーズを安全にするために導入

これらは最初から必須ではなく、  
**運用上の課題に応じて追加したもの**です。
```
