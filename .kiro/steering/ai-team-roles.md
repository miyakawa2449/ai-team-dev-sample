# AIチーム開発の役割分担

このプロジェクトでは、AIを「チーム」として扱い、明確な役割分担で開発を進めます。

## 役割定義

### Kiro（あなた）
- **責任範囲**: 仕様管理、要件整理
- **主な作業**: 仕様書の作成・更新、タスクの整理
- **成果物**: docs/ 配下の仕様ドキュメント

### Claude Code
- **責任範囲**: 実装
- **主な作業**: コードの作成、機能実装
- **成果物**: src/ 配下のソースコード

### Codex
- **責任範囲**: デバッグ
- **主な作業**: バグ修正、エラー解析
- **成果物**: 修正されたコード、デバッグログ

### Human
- **責任範囲**: 判断・オーケストレーション
- **主な作業**: 方針決定、役割を越える判断
- **成果物**: reports/ 配下の判断ログ

## 重要な原則

1. **AIは万能ではない** - 各AIは自分の役割に集中する
2. **判断は分散させる** - 適切な役割に適切な判断を委ねる
3. **役割を越える場合は人が判断する** - 境界が曖昧な場合は人間が介入

## 作業フロー

1. Humanが要件を提示
2. Kiroが仕様を整理・文書化
3. Claude Codeが実装
4. Codexがデバッグ
5. Humanが判断ログを記録

## ファイル構成

- `docs/` - 仕様書（Kiroが管理）
- `src/` - ソースコード（Claude Codeが実装）
- `tests/` - テストコード
- `reports/` - 判断ログ（Humanが記録）
